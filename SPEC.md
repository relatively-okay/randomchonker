# Specification: randomchonker

A single HTML file (`index.html`) with no external dependencies and no build step. All logic is inline. The page displays a fullscreen, auto-advancing slideshow of images fetched from a Reddit subreddit, with deduplication across sessions and a QR code overlay on pause.

---

## Layout

- `<body>` background is always `#000`. No white is ever visible.
- Two absolutely-positioned full-viewport `<div>` elements (frame A and frame B) are stacked on top of each other. Each contains a single `<img>`. This is the A/B buffer for crossfade transitions.
- One `<div id="qr-overlay">` covers the full viewport, used for the pause modal.
- One `<div id="status">` sits fixed at the bottom-center for transient text messages.
- No other visible elements.

---

## Image display

- Images are centered and fit within the viewport using `object-fit: contain`, preserving aspect ratio. No cropping.
- The next image is loaded into the hidden frame before any transition begins. The visible frame never goes blank.
- Transition between images is a 1-second `opacity` crossfade (`ease-in-out`). Both frames are `position: fixed; inset: 0` so they overlap exactly. The incoming frame fades in while the outgoing frame fades out simultaneously.
- The cursor is hidden (`cursor: none` on `body`) while the slideshow is running.

---

## Data source

- Source: `https://www.reddit.com/r/{SUBREDDIT}/hot.json?limit=100&raw_json=1`
- `raw_json=1` prevents Reddit from HTML-encoding URLs in the response.
- Pagination: the `after` token from each response is used to fetch subsequent pages. When the response contains no `after` value, no further fetches are attempted until a loop-around reset occurs.
- Posts are filtered to image-only: include only posts where `post_hint === 'image'` or where `url` matches `/\.(jpe?g|png|gif|webp)(\?|$)/i`, AND `is_video` and `is_gallery` are both false.
- Image URL extraction: prefer `preview.images[0].source.url` (full-resolution Reddit-hosted preview); fall back to `post.url` if it matches the image extension pattern.
- All fetched posts are stored in memory for the lifetime of the page as `{ id, url, title, score }`.

---

## Deduplication

- Seen post IDs are persisted in `localStorage` under the key `chonker_seen_v1` as a JSON array.
- Maximum 400 IDs are retained. When the array exceeds this, the oldest entries are dropped (keep the last 400).
- When selecting a post to display, unseen posts (those whose ID is not in `localStorage`) are preferred. One is chosen by **weighted random selection** where each post's weight is `Math.max(1, post.score)` (its Reddit upvote score, floored at 1). Higher-scored posts are proportionally more likely to be shown.
- If all fetched posts have been seen and more pages are available (`after` token exists), fetch the next page and retry.
- If all fetched posts have been seen and no more pages exist, the seen list is cleared from `localStorage` and selection runs again from the full pool (weighted). A status message "looped around — starting fresh" is shown briefly.

---

## Timing and cycling

- Each photo is displayed for `DURATION` milliseconds (default: 120 000 ms).
- The cycle is driven by `setTimeout` chains, not `setInterval`. Each cycle schedules the next only after completing.
- A boolean `inCycle` flag prevents concurrent cycle executions.
- If no posts are available (fetch failed or still in progress), the cycle retries after 3 000 ms.
- If an image fails to load (`onerror`), it is skipped and the cycle runs again immediately (delay 0).
- When fewer than 15 unseen posts remain in memory and more pages are available, a background fetch is triggered automatically.

---

## Pause and QR overlay

### Pausing
- Clicking anywhere on the page (outside the QR card) while the slideshow is running pauses it: the `setTimeout` is cleared and the QR overlay is shown.
- While paused, the cursor is restored to `default`.

### QR overlay contents
- A semi-transparent dark backdrop (`rgba(0,0,0,0.65)`) covers the full viewport.
- A centered card contains:
  1. A QR code image linking to the current Reddit post.
  2. The post title, clamped to 2 lines.
  3. A small hint label: "tap anywhere to continue".
  4. A "next photo →" button.

### QR code image
- Generated by `https://api.qrserver.com/v1/create-qr-code/` with parameters: `size=210x210`, `color=ffffff`, `bgcolor=000000`, `format=png`, and `data` set to the URL-encoded Reddit post URL.
- Reddit post URL format: `https://www.reddit.com/r/{SUBREDDIT}/comments/{post.id}/`

### Resuming
- Clicking anywhere on the backdrop (outside the card) resumes the slideshow: the overlay hides, the cursor hides, and the timer is rescheduled for the full `DURATION`.
- Clicking "next photo →" skips to the next photo immediately and resumes playing. This click does not propagate to the backdrop handler.

## Keyboard controls

| Key | Behavior |
|---|---|
| `Space`, `ArrowRight`, `ArrowDown`, `Enter` | Skip to next photo immediately; resume if paused |
| `F` / `f` | Toggle browser fullscreen |

---

## Constants (top of `<script>`)

| Name | Default | Description |
|---|---|---|
| `SEEN_KEY` | `'chonker_seen_v1'` | `localStorage` key for seen IDs |
| `MAX_SEEN` | `400` | Maximum seen IDs to retain |
| `DURATION` | `120_000` | Milliseconds per photo |
| `FADE_MS` | `1_000` | Crossfade duration in ms; must match the CSS `transition` value |
| `SUBREDDIT` | `'chonkers'` | Subreddit to fetch from |

---

## Status messages

A `#status` element at the bottom-center of the screen displays transient messages for 3 seconds then fades out. Currently used for: "looped around — starting fresh".

---

## External services used

| Service | Purpose |
|---|---|
| `reddit.com` | Subreddit JSON API for post metadata and image URLs |
| `api.qrserver.com` | QR code image generation |
